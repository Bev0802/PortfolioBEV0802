Sales Accounting System

SalesAccounting - is a system for automating purchase and sale processes designed to manage products, orders, and organizations.

The project consists of two main parts: the server part (productDB) and the client part (wholeSale).

Project structure

1. productDB

The server part is responsible for managing a database containing information about products, orders, documents, employees, and organizations.

Main technologies:
  • Spring Boot: A framework for creating stand-alone, production-grade Spring applications.
  • Spring Data JPA: A tool for interacting with a database.
  • H2 Database Engine: Used as a database during development.
  • PostgreSQL: Used as the main database.
  • Spring Cloud Netflix Eureka: A service for registering and discovering microservices.
  • Spring Security: Provides application security.
  • JUnit: A framework for unit testing.
  • Lombok: Annotations to make code easier to work with.
  • Docker: Support for application containerization.
  • Docker Compose: Container manipulation.
  • Git: Version control system.
  • GitHub: Repository.
Main functions:
  • Product management: creating, updating, deleting, and searching products.
  • Order management: creating, updating, confirming, canceling, and searching for orders.
  • Document management: creating documents based on orders, searching, and deleting documents.
  • Employee management: registration, authentication, cloning, and deletion of employees.
  • Organization management: registration, authentication, updating, and deleting organizations.

2. wholeSale

The client part provides a web interface for interacting with the system.

Main technologies:
  • Spring Boot: A framework for creating stand-alone, production-grade Spring applications.
  • Thymeleaf: A template engine for creating dynamic web pages.
  • Spring Cloud Netflix Eureka: A service for registering and discovering microservices.
  • JUnit: A framework for unit testing.
  • Lombok: Annotations to make code easier to work with.
  • Docker: Support for application containerization.
Main functions:
  • Authentication and registration of employees and organizations.
  • Product management: create, update, delete, and view products.
  • Order management: create, update, confirm, cancel, and view orders.
  • Document management: view documents by buyer and seller.
  • Employee management: view and edit employee information.
  • Organization management: viewing and editing information about organizations.

You can see my code on GitHub: salesAccounting
